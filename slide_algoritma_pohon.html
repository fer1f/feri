Decision Tree

Algoritma machine learning untuk klasifikasi dan regresi dengan struktur seperti pohon.

Komponen Utama:

  • Root Node - Titik awal, mewakili seluruh dataset
  • Decision Node - Keputusan berdasarkan fitur tertentu
  • Leaf Node - Hasil akhir (label kelas atau nilai numerik)

Parameter Utama Decision Tree

  • Criterion: Gini Impurity, Entropy, Log Loss
  • max_depth: Kedalaman maksimal pohon
  • min_samples_split: Minimum sampel untuk split
  • min_samples_leaf: Minimum sampel pada leaf
  • max_features: Jumlah fitur untuk pembagian
  • Splitter: Best atau Random

Kelebihan & Kekurangan Decision Tree

Kelebihan ‚úì

  • Mudah dipahami dan diinterpretasikan
  • Memerlukan sedikit pra-pemrosesan data

Kekurangan ‚úó

  • Rentan terhadap overfitting
  • Pohon dalam menjadi kompleks dan sulit diinterpretasikan

Random Forest

Ensemble learning yang menggabungkan banyak Decision Trees untuk meningkatkan akurasi dan mengurangi overfitting.

Konsep Utama:

  • Menggunakan bagging (bootstrap aggregating)
  • Setiap pohon dilatih dengan subset acak dari data dan fitur
  • Hasil akhir: voting (klasifikasi) atau rata-rata (regresi)
  • Lebih robust dan tidak mudah "menghafal" data latih

Parameter Utama Random Forest

  • n_estimators: Jumlah pohon (default: 100, rekomendasi: 100-300)
  • max_depth: Kedalaman maksimal (biasanya: 10-20)
  • min_samples_split: Minimum sampel untuk split (default: 2)
  • min_samples_leaf: Minimum sampel di leaf (1-5 untuk dataset besar)
  • max_features: Fitur per split ("sqrt" atau "log2")
  • bootstrap: Sampling dengan penggantian (default: True)

Kelebihan & Kekurangan Random Forest

Kelebihan ‚úì

  • Akurasi tinggi
  • Robust terhadap overfitting
  • Menangani data tidak seimbang
  • Menangani data hilang dengan baik

Kekurangan ‚úó

  • Kebutuhan memori tinggi
  • Interpretabilitas rendah
  • Lambat pada prediksi
  • Waktu pelatihan lama

XGBoost (Extreme Gradient Boosting)

Implementasi populer dari Gradient Boosting - metode ensemble yang membangun model secara sekuensial.

Perbedaan dengan Random Forest:

Random Forest: Bagging (paralel) ‚Üí Kurangi variansi
XGBoost: Boosting (sekuensial) ‚Üí Kurangi bias

Gradient: Optimasi gradient descent untuk meminimalkan error

Cara Kerja XGBoost

  • Model pertama memprediksi target data
  • Error dari model pertama dihitung
  • Model baru ditambahkan dengan bobot (learning rate) untuk memperbaiki prediksi
  • Proses diulang hingga error cukup kecil atau mencapai iterasi maksimum
  • Prediksi akhir = jumlah kontribusi seluruh pohon

Parameter Utama XGBoost

  • n_estimators: Jumlah pohon sekuensial
  • learning_rate (eta): Kontribusi tiap pohon (0.05-0.1)
  • max_depth: Kedalaman maksimal (3-6)
  • subsample: Proporsi data pelatihan (0.5-1)
  • colsample_bytree: Proporsi fitur per pohon
  • gamma: Minimum pengurangan loss untuk split
  • reg_lambda/reg_alpha: Regularisasi L2/L1
  • objective: Fungsi loss (reg:squarederror, binary:logistic)

Kelebihan & Kekurangan XGBoost

Kelebihan ‚úì

  • Akurasi sangat tinggi
  • Regularisasi untuk kurangi overfitting
  • Menangani nilai hilang
  • Paralelisasi dan komputasi terdistribusi
  • Early stopping

Kekurangan ‚úó

  • Kompleks untuk tuning
  • Waktu pelatihan bisa lama
  • Interpretabilitas menurun

Optuna - Hyperparameter Tuning

Library open-source untuk optimasi hyperparameter yang efisien dan otomatis.

Fitur Utama:

  • Define-by-run: Pencarian dinamis saat eksekusi
  • Bayesian Optimization: TPE (Tree-structured Parzen Estimator)
  • Pruning: Hentikan percobaan yang tidak menjanjikan
  • Integrasi dengan XGBoost, LightGBM, PyTorch
